Overview

Commerce is a robust, Django-based auction platform designed to facilitate a seamless online bidding experience. It empowers users to list items for auction, place real-time bids, engage with the community through comments, and track their favorite items via a watchlist.

Built with scalability and user experience in mind, this project demonstrates a clean separation of concerns using the Django MVT (Model-View-Template) architecture.

Key Features

Listing Management

Users can create rich auction listings complete with titles, descriptions, starting bids, categories, and image URLs. Owners have the ability to close auctions at any time.

Bidding System

Dynamic bidding engine allowing users to place competitive bids. The system validates bids to ensure they exceed the current highest offer.

Watchlist

Personalized watchlist feature enabling users to track specific auctions they are interested in without placing an immediate bid.

Categorization

Robust filtering system allowing users to browse items by specific categories (e.g., Electronics, Fashion, Toys).

Comments & Interaction

Social features allowing authenticated users to discuss listings, ask questions, or provide feedback directly on the item page.

Admin Dashboard

Full-featured Django Admin interface for site administrators to manage users, listings, bids, and comments efficiently.

Architecture & Data Models

The application relies on a relational database schema optimized for data integrity and efficient querying. Below are the core models defining the system.

User

Extends AbstractUser.

  • watchlist: M:N relationship with Item. Tracks watched items.

Item

The core entity representing an auction listing.

  • title: CharField (64 chars).
  • description: TextField.
  • starting_bid: DecimalField.
  • isopen: Boolean. Auction status.
  • category: ForeignKey to Category.
  • owner: ForeignKey to User.

Bid

Records a monetary offer on an item.

  • amount: DecimalField.
  • item: ForeignKey to Item.
  • bidder: ForeignKey to User.
  • timestamp: DateTimeField (auto-add).

Comment

User interactions on listings.

  • text: Content of the comment.
  • item: ForeignKey to Item.
  • by: ForeignKey to User.

Category

Classification for items.

  • name: CharField. Category name.

Getting Started

Follow these steps to set up the development environment locally.

Prerequisites

Python 3.x Django

Installation

  1. Clone the repository:

    git clone <repository-url>
cd commerce

  2. Install dependencies:

    pip install -r requirements.txt

  3. Apply database migrations:

    python manage.py makemigrations
python manage.py migrate

  4. Run the development server:

    python manage.py runserver

Access the application at http://127.0.0.1:8000/.
